维护说明

处理应用后，记得同步维护这三处

为了保证 GitHub Pages 首页、仓库文档和应用入口始终一致，后续每新增、修改或下线一个应用，都要同步更新导航页、README.md 和 AGENTS.md。

1. 更新导航页

在根目录 index.html 中补充应用数据，让首页卡片、统计和快速入口保持最新。

2. 更新 README

补充应用名称、目录、功能说明、Pages 路径和当前状态，便于仓库协作与检索。

3. 更新 AGENTS

如果本次任务补充了新的维护要求、交付规则或目录约束，要同步写回根目录 AGENTS.md。

本页为仓库首页导航。应用均采用相对路径跳转，适配 GitHub Pages 项目页部署。

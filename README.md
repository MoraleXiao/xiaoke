# xiaoke

一个面向课堂教学场景的纯 HTML 小应用仓库。

本仓库中的每个应用都以独立目录存在，并通过 GitHub Pages 按路径进行访问，适合做课堂演示、互动练习、小游戏或轻量工具。

## 仓库目标

- 用最简单的技术栈制作可直接打开使用的课堂小应用。
- 每个应用一个目录，彼此独立，便于维护和按路径访问。
- 所有应用默认面向 GitHub Pages 静态部署，不依赖服务端。
- 每次新增或修改应用时，同步维护本页的应用清单与功能说明。

## 目录约定

推荐结构如下：

```text
.
├── AGENTS.md
├── README.md
├── app-name-a/
│   ├── index.html
│   ├── style.css
│   ├── script.js
│   └── assets/
├── app-name-b/
│   └── index.html
└── shared/
```

说明：

- 每个应用都放在仓库根目录下的独立目录中。
- 每个应用目录至少应包含一个 `index.html`。
- 目录名应简洁、稳定，建议使用短横线命名，便于直接映射为 URL 路径。
- 若没有明确复用需求，不要提前抽取公共层或引入构建工具。

## GitHub Pages 访问规则

如果仓库按 GitHub Pages 默认项目页方式发布，访问形式通常为：

- 仓库首页：`https://moralexiao.github.io/xiaoke/`
- 单个应用：`https://moralexiao.github.io/xiaoke/<应用目录>/`

例如应用目录为 `fraction-trainer/`，则访问地址通常为：

`https://moralexiao.github.io/xiaoke/fraction-trainer/`

为保证部署正常：

- 页面内资源优先使用相对路径。
- 不要假设站点部署在域名根路径 `/`。
- 不要依赖后端接口或本地开发服务器才能运行。

## 新增应用时的最小要求

新增一个应用时，至少满足以下条件：

1. 在仓库根目录创建独立应用目录。
2. 提供可直接访问的 `index.html`。
3. 若需要脚本或样式，可增加 `script.js`、`style.css`。
4. 若需要图片、音频等资源，放入应用目录内的 `assets/`。
5. 在下方“应用列表”中补充该应用的目录、功能和访问路径。

## 维护流程

后续无论由人还是 AI 维护本仓库，都应遵循以下流程：

1. 先调研仓库现状和需求，避免重复建设或误改结构。
2. 先给出方案，再开始实现。
3. 实现完成后，更新本 `README.md` 中的应用列表和功能说明。
4. 确认内容可用于 GitHub Pages 访问。
5. 默认将完成结果合并到 `main` 分支并推送远端。
